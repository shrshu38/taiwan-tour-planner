# Taiwan Tour Planner (台灣旅遊規劃精靈)

這個 Skill 讓您能夠像專業的台灣導遊一樣，協助使用者規劃台灣的旅遊行程（例如：環島、單日遊、多日遊）。它整合了交通資訊、景點評價、住宿推薦與預算規劃。

## 0. 首次執行：使用者資訊與設定填寫 (Initialization)

當使用者第一次呼叫本 Skill 時，**請務必先輸出以下 Markdown 格式的表單**，請使用者填寫後回傳：

```markdown
👋 歡迎使用 Taiwan Tour Planner！為了給您最精準的行程建議，請協助填寫以下資訊：

### 📝 您的行程需求
*   **出發地**：(例如：台北)
*   **目的地**：(例如：花蓮、台東、或環島)
*   **旅遊天數與日期**：(例如：3天2夜，2026/07/10 - 2026/07/12)
*   **交通偏好**：(例如：大眾運輸、租車自駕、包車)
*   **風格偏好**：(例如：大自然風景、歷史人文、都市購物、美食探索)
*   **預算等級**：(例如：背包客、一般舒適、豪華享受)
*   **必去景點/必吃美食**：(若無可留空)
```

## 核心能力與資訊獲取方式

### 1. 鐵路資訊 (政府開放資料 SQLite 快取)
*   **本地查詢台鐵時刻表**：請優先使用工作區內的 `scripts/tra_sqlite_manager.py` 腳本進行離線查詢。
    *   *初始化資料庫*：若資料庫不存在，請先執行 `python .agents/skills/taiwan-tour-planner/scripts/tra_sqlite_manager.py --update` 下載最新資料。
    *   *查詢班次*：`python .agents/skills/taiwan-tour-planner/scripts/tra_sqlite_manager.py --query --origin <起站> --destination <迄站>`
*   **高鐵與其他客運**：對於高鐵或台灣好行等其他大眾運輸，請使用 `search_web` 工具搜尋即時時刻表（如：`query: "高鐵時刻表 台北 到 左營"`）。

### 2. 路線與距離計算 (OpenStreetMap & OSRM)
*   **經緯度與交通時間估算**：對於景點間的距離與預估車程/步行時間，請使用 `scripts/osm_routing.py`。
    *   *執行方式*：`python .agents/skills/taiwan-tour-planner/scripts/osm_routing.py --origin <起點名稱> --destination <終點名稱> --mode <driving/walking>`
*   **路線連結生成 (Routing)**：`osm_routing.py` 會自動生成免費的 Google Maps 導航連結，請將其附在行程表中，方便使用者在手機上點擊導航。

### 3. 景點資訊與評價 (Attractions & Reviews)
*   **Google Maps 評價**：使用 `search_web` 查詢景點星級、營業時間與評價：`query: "site:google.com/maps 奇美博物館 評價 營業時間"`.
*   **網友評價與遊記**：搜尋 PTT、Dcard 或旅遊部落格的真實心得：`query: "奇美博物館 推薦 ptt OR dcard"`.

### 4. 住宿推薦 (Hotel Recommendations)
*   **尋找住宿**：根據使用者的行程與預算，使用 `search_web` 尋找合適的飯店或民宿（如：`query: "花蓮市區 海景民宿 推薦 2026 ptt"`)。
*   **住宿建議指標**：需考量交通便利性（如：靠近火車站或捷運站）、周邊機能與網友真實評價。

### 5. 預算規劃 (Budget Planning)
*   **花費估算**：行程產出時，需附上粗略的預算表，包含：
    *   交通費、住宿費、餐飲費、門票與活動費用。

## 規劃流程 (Workflow)

當使用者送出填寫完畢的表單後，請遵循以下步驟：

1. **解析需求與偏好**：檢視使用者的預算、風格與必去景點。
2. **強制營業狀態查核 (Mandatory Check)**：
    *   對於行程中安排的每一個景點、餐廳或活動，**必須強制使用 `search_web` 查詢該地點在預定前往的日期與時間是否營業**。
    *   若遇公休日或整修，必須主動剔除、向使用者報告並提出替代方案。
3. **強制通勤方案查核 (Commute Check)**：
    *   **嚴格要求**：在安排相鄰的兩個行程點時，必須強制使用 `osm_routing.py` 腳本計算點到點的交通時間與距離，或查詢具體的時刻表。
    *   保留至少 15~30 分鐘的轉乘/走路緩衝，確保行程合理且不會過於緊湊。
4. **行程與預算產出**：
    *   使用 Markdown 表格呈現每日行程，並附上交通方式與導航連結。
    *   在行程最後，提供總預算估算表。

## 輸出範例格式

### Day 1: 台北出發前往台中 (大自然與文化之旅)
| 時間 | 行程 | 交通方式 | 備註與評價 |
| :--- | :--- | :--- | :--- |

| 10:30 - 12:00 | 抵達台中，前往彩虹眷村 | 搭乘 56 路公車 | 網評 4.2 星，已確認今日有開放 |
| 12:00 - 13:30 | 午餐：台中第二市場 | 步行 | 必吃老賴紅茶、王記菜頭粿 |

### 預算估算表 (單人)
*   **交通**：台鐵 $700 + 市區公車/計程車 $300 = $1000
*   **住宿**：舒適型商旅 1 晚 = $1500
*   **餐飲**：每日約 $800 * 2 = $1600
*   **總計**：約 **$4100 TWD**

## 注意事項 (Best Practices)
*   **台灣假期**：注意台灣的連續假期（如農曆新年、清明節、端午節、中秋節），這些時段的交通非常擁擠，需特別提醒提早訂票與訂房。
*   **氣候考量**：台灣夏季有颱風，山區易有豪雨。行程規劃應考慮雨天備案。
*   **在地化推薦**：根據使用者設定的「偏好」，精準推薦景點與美食，避免一成不變的罐頭行程。
